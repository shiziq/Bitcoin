Competitive ContextRivaling M-Pesa

M-Pesa charges a percentage-based fee per transaction. Send KSh 10,000 and you pay KSh 100–300. Bato flips the economics: every USDT transfer costs approximately KSh 0.15 — a flat blockchain fee, same for KSh 500 or KSh 50,000.

A Kenyan business processing KSh 500,000 monthly through M-Pesa pays ~KSh 5,000–15,000 in fees. On Bato: ~KSh 500 in blockchain fees plus 2% spread on loads — a commercially transformative cost structure.

What the Cashier Sees

Shared Code — Mode B

While Jane scans, the POS displays the shared 6-character code A7K3M9. Jane verifies the code on BOTH screens — POS and Authenticator — enters it, and approves. This prevents MITM attacks: both parties see the same code. 60 seconds TTL, 3 entry attempts maximum.

Phase 7A — Load

Jane opens KCB app, selects "Send to Bracelet." Enters 50 USDT. QR code appears. She brings the bracelet near her phone's NFC reader. Bracelet reads the NDEF payload. Authenticator confirms: "Load 50 USDT to Bracelet #A3K9?" Jane approves.

Phase 7B — Bracelet Tap at POS: NFC Reader + Shared Code Verification

How It Works

Cashier rings KSh 650. Jane taps bracelet on NFC reader. Authenticator push: "Pay KSh 650 to Naivas? Code: A7K3M9." Jane verifies code matches POS, enters code, approves. 2-second settlement.

Security

Same device key model as the phone. Factory-provisioned key in tamper-resistant NTAG 424 DNA chip. Every NFC transmission signed. Bracelet deactivatable from app if lost.
